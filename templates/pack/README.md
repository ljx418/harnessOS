# Pack Template

This directory is a V3.5-G scaffold for creating new Domain Packs. It is not a
runtime pack instance and must not be auto-discovered by `pack.list`.

To instantiate a pack manually:

1. Copy this directory to `packs/<new_pack>` or to a test fixture directory.
2. Edit `manifest.json`.
3. Add the parent pack root to `HARNESS_PACK_PATHS`, `HARNESS_EXTERNAL_PACK_PATHS`,
   an `AppProfile.pack_paths` entry, or a test-specific `PackRegistry.load_from_paths(...)`.

This phase does not provide a `harnessos pack init` CLI.

Manifest compatibility rules:

- Keep `name`, `domain`, and `version` as the identity contract.
- Use `manifest_schema_version`, `min_harnessos_version`, and
  `target_harnessos_version` for compatibility checks.
- Use `policy_bundles`, not `policies`.
- Use `artifact_kinds` and `artifact_schemas`, not `artifact_types`.
- Compatibility warnings are generated by the loader or assembly result; do not
  declare `compatibility_warnings` as manifest truth.
