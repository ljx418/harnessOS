# 工位产物：产品规划

## 1. 目标理解

作为产品规划工位，我们的核心职责是将“为 HarnessOS 设计并验证一个可审计的工程任务执行流程”这一宏观目标，转化为具备可操作性的产品需求与业务逻辑。

**核心要点解读：**
*   **Auditability (可审计性)**：这是平台的核心竞争力。要求系统不仅执行任务，还要完整记录“谁、在什么时候、因为什么、执行了什么、结果如何”。这是所有功能设计的基石。
*   **Headless-first (无头优先)**：意味着主要交互发生在 API/CLI 层，但因包含“人工确认”步骤，必须存在清晰、高效的审批管理前台(Frontend/Console)。
*   **Constraint-Driven (受限于约束)**：“不允许自动修改代码”和“用户确认制”是铁律。这定义了平台的本质是 **"Governance Platform" (治理平台)** 而非 purely "Automation Platform" (自动化平台)。

## 2. 输入引用

*   **目标**：为 HarnessOS 的 Headless-first 工作流平台设计并验证一个可审计的工程任务执行流程。
*   **核心约束**：
    *   **安全红线**：不允许 Agent 自动修改代码；不允许自动发布。
    *   **交互模式**：所有持久操作都必须用户确认 (Human-in-the-loop)。
    *   **工程规范**：每个阶段必须产出 artifact；失败阶段必须可重跑并保留 attempt history。
