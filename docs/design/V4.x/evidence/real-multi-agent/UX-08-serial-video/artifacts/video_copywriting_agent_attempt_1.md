# 文案 Agent 工位产物

## 目标理解

基于 HarnessOS Headless 发布视频 Brief，本工位负责生成**视频旁白、标题和短视频文案**，用于向目标受众（研发负责人、技术管理者、业务用户）传达产品核心价值，重点突出：

- Headless-first 架构转型
- 串行 Agent 工位协作模式
- 工作流可审计、可恢复、可解释的特性

## 关键输出

### 视频标题

> **"从代码到掌控：HarnessOS Headless 工作流引擎发布"**

备选标题：
- "AI 工作流的透明之道：HarnessOS Headless 来了"
- "把 AI 工作流装进透明盒子"

### 视频旁白文案

**【开场 0:00-0:15】**
> 你们有没有遇到过这种情况？AI 工作流跑了三个小时，结果出来一堆不知道哪儿来的错误，想查日志都不知道从哪下手。

**【引入 0:15-0:45】**
> 今天，HarnessOS 4.x 带来了全新的答案——Headless 工作流引擎。
>
> 你可以用自然语言描述需求，或者用 TUI 拖拽式操作，系统会自动生成标准化的 WorkflowSpec、清晰的 Drawio 流程图，还有完整的 HTML 运行报告。
>
> 每一个工位产出了什么、用了多少 token、质量如何，全部一目了然。

**【核心卖点 0:45-1:30】**
> 更重要的是，这不是黑盒。
>
> 来看这个串行多 Agent 工作流：文案 Agent 生成脚本 -> 图像 Agent 生成配图 -> 审核 Agent 检查合规。如果中间的图像生成出了问题，只需要重跑图像 Agent，下游自动标记为 stale 状态，等你修复后继续运行。全程有据可查。
>
> 所有持久变更都需要你确认。Agent 只提建议，绝不自行执行。

**【收尾 1:30-2:00】**
> 现在，无论是研发负责人想搭建工作流，技术管理者想审计过程，还是业务用户想查看质量报告，都有了统一的入口。
>
> HarnessOS Headless，让 AI 工作流真正可控。

## 质量风险

| 风险类型 | 说明 | 缓解建议 |
|---------|------|---------|
| 技术术语过载 | 受众包含业务用户，部分术语可能难以理解 | 将 "WorkflowSpec"、"stale" 等术语放在演示画面中，用视觉语言辅助说明 |
| 时长把控 | 旁白约 2 分钟，需配合画面节奏 | 建议语速控制在每分钟 150 字以内，预留画面展示时间 |
| 核心信息遗漏 | 必须桥段需全部出现，注意逻辑衔接 | 按 "痛点引入 → 方案介绍 → 功能演示 → 价值收尾" 结构组织，确保桥段自然嵌入 |
