# 工位产物：发布准备 Agent

## 目标理解

本工位位于串行多 Agent 视频工作流的最终阶段，负责将上游 工位（创意脚本 Agent、视觉设计 Agent、配音制作 Agent 等）产出的素材整合为可直接发布的视频成片及配套交付文档。

**核心目标**：

1. 生成符合 Brief 要求的最终视频成品，确保所有桥段完整呈现
2. 输出发布清单，明确交付内容及版本信息
3. 提供交付说明，指导后续使用方如何应用该视频及工位产物

**与Brief的对应关系**：

- Brief 要求"现代 SaaS 产品介绍，节奏清晰" → 本工位需确保视频节奏剪辑符合 SaaS 产品介绍范式
- Brief 规定必须出现的桥段 → 本工位需验证最终成片中四个桥段是否完整且逻辑连贯
- Brief 强调"可审计、可恢复、可解释" → 交付物需包含可追溯的制作过程记录

## 关键输出

### 1. 视频成片

| 文件名 | 格式 | 分辨率 | 时长 |
|--------|------|--------|------|
| harnesCos_v4_headless_intro.mp4 | H.264 MP4 | 1920x1080 | ~3-5分钟 |
| harnesCos_v4_headless_intro.webm | VP9 WebM | 1920x1080 | ~3-5分钟 |

### 2. 发布清单

```
发布清单_v1.0.md
├── 视频文件
│   ├── mp4 (主分发格式)
│   └── webm (Web兼容格式)
├── 截图/PDF预览
│   ├── key_frames.pdf (关键帧PDF汇总)
│   └── thumbnailSprite.jpg (缩略图精灵)
├── 元数据
│   ├── video_metadata.json (时长、分辨率、码率、字幕轨道)
│   └── transcript.srt (完整字幕文案)
└── 证明材料
    ├── evidence_chain.json (各工位产出追溯链)
    └── audit_log.md (制作过程审计日志)
```

### 3. 交付说明

包含以下内容：

- 视频基本参数说明
- 各桥段时间码索引（方便快速定位）
- 推荐发布渠道及配置建议
- 定制化修改指引（如需替换Logo、联系方式等）

## 输入引用

本工位依赖以下上游工位的关键产出：

1. **创意脚本 Agent**：最终版解说词文案（含旁白稿、分镜描述）
2. **视觉设计 Agent**：动画工程文件、Drawio 工作流图、图表素材
3. **配音制作 Agent**：旁白音频文件（MP3/WAV）、音效素材
4. **后期合成 Agent**：AE/PR 工程文件、最终混音输出

**必须核验的输入项**：

- 四个必须桥段在时间轴上的位置准确性
- Headless-first 路线展示与配音文字的一致性
- 中间工位重跑和下游 stale 状态的视觉表现
- HTML 报告和治理证据链的截图/动画

## 质量风险

| 风险类型 | 描述 | 严重程度 | 缓解措施 |
|----------|------|----------|----------|
| 桥段遗漏 | 四个必须桥段中有任何一段未在成片中完整呈现 | 高 | 使用时间码清单逐项核对 |
| 音画不同步 | 配音与关键画面切换的时间点偏差超过±2帧 | 中 | 最终输出前进行多端播放器校验 |
| 分辨率不一致 | 不同素材源的分辨率混用导致输出降质 | 低 | 统一按 1920x1080 规格重新渲染 |
| 元数据缺失 | 缺少字幕轨道或Metadata不准确 | 中 | 批量检查工具复核 JSON 结构完整性 |
| 审计链断裂 | evidence_chain.json 无法追溯至最早工位 | 高 | 回溯检查每个工位的产物哈希值连续性 |
