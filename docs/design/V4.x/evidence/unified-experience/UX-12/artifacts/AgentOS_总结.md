# AgentOS 技术文档总结

## 内容概览

AgentOS 采用 **proposal-first（提案优先）** 的工作流控制台架构。系统通过画布展示草案、运行状态和治理证据，所有真实修改必须经过 BFF（Backend For Frontend）的受控路径。工作流由多个专业化节点组成，每个节点暴露可观察状态和产物引用。

---

## 核心主题

| 主题 | 说明 |
|------|------|
| **BFF 职责** | 负责权限验证、治理规则执行和 DTO redaction（数据脱敏） |
| **Runtime 职责** | 执行工作流节点，管理任务调度 |
| **Agent 定位** | 仅生成建议和交接，不能自动执行 mutation（变更操作） |
| **用户确认机制** | 所有持久修改都需要用户明确确认 |

---

## 关键知识点

### 架构设计原则
- **EventBridge 限制**：只触发 refresh（前端刷新），前端不能从事件 payload 构造 truth（真相数据）
- **Mutation 控制**：Agent 助手只能生成建议和交接，不能直接执行变更

### 工作流节点类型

| 节点 | 功能 |
|------|------|
| 输入节点 | 收集用户授权 |
| 扫描节点 | 仅枚举文件结构，不做深度处理 |
| 解析节点 | 解析文件内容 |
| 分组节点 | 对文件进行分组 |
| 总结节点 | 生成 Markdown artifact（产物） |
| 质量检查节点 | 记录未支持文件和空文件夹 |
| 产物发布节点 | 发布最终产物 |
