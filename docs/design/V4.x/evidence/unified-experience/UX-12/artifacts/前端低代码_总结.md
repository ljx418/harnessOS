# 前端低代码技术总结

## 内容概览

前端低代码平台提供可视化的节点编排能力，通过画布设计和节点库两大核心模块实现工作流的可视化配置。画布负责节点的编排、连线和检查，节点库则提供受控的组件目录，两者协同构成完整的低代码开发体验。

## 核心主题

- **画布设计**：可视化节点编排与交互
- **节点分类体系**：输入节点、Agent 节点、治理节点的职责划分
- **状态管理机制**：pending 节点与 proposal 状态的展示
- **数据流架构**：catalog 驱动的受控渲染模式

## 关键知识点

| 知识点 | 说明 |
|---------|------|
| pending 节点 | 用于展示 proposal 状态的过渡节点 |
| Inspector typing | 仅更新本地 dirty state，不触发后端请求 |
| Patch Diff | 点击生成 Patch 后才访问 BFF 进行差异比对 |
| 受控 catalog | 前端仅渲染目录数据，不可自定义运行时语义 |
| 输入节点 | 负责接收文件夹或用户需求作为输入源 |
| Agent 节点 | 负责生成、分析、审查等智能处理任务 |
| 治理节点 | 负责审批、质量检查和审计等管控流程 |
