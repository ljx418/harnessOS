# PRD: harnessOS V4.0 Workflow Studio + Agent 工作流助手

版本：v0.2  
文档状态：V4.0 Workflow Studio 前端体验验收基线。  
适用范围：Stitch 原型生成、产品评审、后续 Codex 前端实现和体验验收。  
原型项目：`https://stitch.withgoogle.com/projects/10240451325799222489`

## 1. 产品定位

### 1.1 产品名称

```text
harnessOS 工作流 Studio
harnessOS Workflow Studio
```

### 1.2 一句话定位

harnessOS 工作流 Studio 是一个现代低代码 AI 工作流开发平台，用户既可以通过节点画布手动编排工作流，也可以通过 Agent 对话框用自然语言生成、修改、优化工作流。

## 2. 核心体验目标

当前页面必须突出三个核心能力：

1. 低代码节点画布。
2. Agent 自然语言工作流助手。
3. Patch / Diff / 用户确认机制。

页面不是普通低代码编辑器，也不是普通聊天机器人，而是：

```text
节点画布 + Agent 协作编辑 + 可审计 Patch 修改
```

用户应该能看到：

- 我可以拖拽节点搭流程。
- 也可以直接告诉 Agent：“帮我生成一个视频制作工作流”。
- Agent 会生成工作流草案或修改建议。
- 系统会展示 Patch Diff。
- 用户确认后才 Apply / Publish。

## 3. 当前原型必须修正的问题

1. Agent 对话框必须明显，不能缺失。
2. 页面不能只是普通低代码画布，必须体现“自然语言调整工作流”。
3. Agent 和画布必须形成联动。
4. Patch Diff / 修改建议必须突出。
5. 用户确认机制必须清楚，不能让用户误以为 Agent 自动改流程。

## 8. 右侧 Agent 工作流助手

这是本次原型最重要的区域。

区域标题：

```text
Agent 工作流助手
Agent 助手
```

不要叫普通“聊天”。

### 8.1 Agent 助手顶部摘要

Agent 助手顶部显示当前上下文：

- 当前工作流：AI 短视频生成工作流。
- 当前选中节点：分镜生成 Agent。
- 当前问题：质量分 0.64，低于阈值 0.8。
- 建议状态：需要优化。

### 8.2 对话内容示例

用户输入：

```text
帮我生成一个适合 60 秒科幻短片的视频制作工作流。
```

Agent 回复：

```text
我将为你生成一个 VideoStudio 工作流，包含：
需求输入、剧情大纲、脚本生成、分镜生成、图像提示词、视频渲染、质量评估、人工审批和发布输出。
```

Agent 操作卡片：

- 生成工作流草案。
- 查看节点列表。
- 应用到画布。

### 8.3 自然语言修改工作流示例

用户输入：

```text
分镜质量太低，帮我优化这个节点。
```

Agent 回复：

```text
我发现「分镜生成 Agent」质量分为 0.64，低于阈值 0.8。
主要问题是角色一致性不足、镜头衔接不自然。
我建议你选择以下方案之一。
```

Agent 显示三个建议卡片：

1. 重跑本工位：不覆盖旧结果，生成新的 `storyboard_v2.json`。按钮：查看建议。
2. 优化 Prompt：生成一个 Patch，增强角色一致性和镜头转场要求。按钮：查看 Diff。
3. 增加质量检查节点：在分镜后增加角色一致性检查节点。按钮：生成 Patch。

### 8.4 Agent 输入框

右侧底部必须有明显输入框：

```text
请输入你的需求，例如：帮我优化分镜节点，增加人工审批。
```

发送按钮：

```text
发送
```

快捷按钮：

- 生成工作流。
- 优化当前节点。
- 诊断失败原因。
- 生成 Patch。
- 解释当前流程。

## 9. Patch Proposal / Diff 面板

当用户点击「查看 Diff」时，右侧或画布旁展示 Patch Diff 卡片。

Patch Proposal 卡片内容：

```text
Patch Proposal

操作：update_station_prompt
目标节点：分镜生成 Agent

Before:
生成分镜描述

After:
生成包含角色一致性、镜头转场、时长控制和场景连续性的分镜描述

Risk Flags:
prompt_change
quality_rule_affected

状态：
等待用户确认
```

操作按钮：

- 查看 Diff。
- 应用到草稿。
- 取消。

如果当前阶段不允许真实 apply，可以显示：

```text
应用到草稿（后续阶段）
```

但 UI 语义必须表达：

```text
Agent 只能提出建议，用户确认后才能应用。
```

## 12. 页面必须体现的核心用户故事

### 用户故事 1：自然语言生成工作流

用户在 Agent 助手中输入：

```text
帮我生成一个 60 秒科幻短片的视频制作工作流。
```

Agent 生成工作流草案。画布出现：

```text
需求输入 -> 剧情大纲 -> 脚本生成 -> 分镜生成 -> 视频渲染 -> 质量评估 -> 人工审批 -> 发布输出
```

### 用户故事 2：自然语言优化节点

分镜生成节点质量分只有 0.64。用户输入：

```text
帮我优化分镜节点。
```

Agent 解释问题，并给出：

- 重跑本工位。
- 优化 Prompt。
- 增加质量检查节点。

### 用户故事 3：自然语言生成 Patch

用户输入：

```text
把分镜节点的质量阈值提高到 0.9，并增加角色一致性检查。
```

Agent 生成 Patch Proposal。系统显示 Diff：

- Before threshold 0.8。
- After threshold 0.9。
- 新增角色一致性检查节点。

用户确认后才能 Apply。

### 用户故事 4：用户确认，不让 Agent 静默修改

- Agent 不能直接修改发布版本。
- Agent 只能生成建议和 Diff。
- 用户点击 Apply to Draft 后才修改草稿。
- 用户点击 Publish Version 后才发布新版本。

## 13. 明确禁止的界面误导

页面中不要出现：

- Agent 已自动修改工作流。
- Agent 已自动发布版本。
- 一键自动应用。
- 自动上线。
- 自动部署。

推荐表达：

- Agent 已生成修改建议。
- 请查看 Diff。
- 等待用户确认。
- 应用到草稿。
- 发布新版本。

## 14. Stitch 目标页面

后续 Stitch 原型至少需要覆盖 4 张图：

1. Workflow Studio 主页面：左侧节点库、中央画布、右侧 Agent 工作流助手、底部运行面板。
2. 自然语言生成工作流：Agent 输入 60 秒科幻短片需求，Agent 回复并生成工作流草案，画布出现节点。
3. 自然语言优化节点：分镜生成节点质量 0.64 warning，Agent 给出重跑、优化 Prompt、增加质量检查节点。
4. Patch Diff 确认页：Patch Proposal、Before / After、Risk Flags、等待用户确认、Apply to Draft、Reject。

## 15. 当前实现验收边界

当前 V4.0 前端仍属于 dev/local UI shell：

- 可以展示 Agent 工作流助手、自然语言修改建议和 Patch Diff 确认语义。
- 可以展示 VideoStudio 示例工作流和分镜质量 warning。
- 不能声明完整 Workflow Studio ready。
- 不能声明 AgentTalkWindow ready。
- 不能声明 production-ready external app support。
- 不能让 Agent 静默 apply / publish。
