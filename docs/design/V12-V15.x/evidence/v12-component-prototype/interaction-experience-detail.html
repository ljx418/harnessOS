V12-0P 输入 · 交互体验细化规格

把 Light Studio 从“视觉方向”推进到“可实现、可验收的交互合同”

本页围绕左侧一二级导航、画布高级交互、面板层级/Z 轴、设计语言一致性和下方工作台 tab 做具体 UX 约束。 它是 V12-0P 高保真原型输入，不是浏览器实现或运行证据。

2. 画布交互体验

高级感来自可预期的反馈：曲线、吸附、惯性、端口高亮、禁用原因和不静默失败。

canvas
1hover

节点抬高，端口出现，邻接边 30% 高亮。

2click

蓝色描边，右侧检查器 180ms 内同步。

3drag

显示 ghost transform 和吸附线；V12 不提交持久变更。

4connect

从输出端口拖出曲线，合法目标发光。

5invalid

红/琥珀曲线回弹，toast + inspector 说明原因。

6audit

点击证据 ref，高亮相关节点并打开证据 tab。

V12 画布仍是 read-model；拖拽、连线、自动布局只能在高保真原型中展示交互预期，不可被写成已经实现可编辑 Studio。

4. 设计语言一致性

同一状态、同一动作、同一边界必须在 topbar、节点、检查器和下方工作台里保持一致。

tokens
颜色
  • 蓝色：选中和主操作
  • 绿色：完成 / 在线 / 证据就绪
  • 琥珀：等待确认 / 阻断 / 缺证据
  • 红色：失败 / 拒绝 / 危险
形态
  • 8px 控件半径
  • 12px 卡片半径
  • 16px 大面板半径
  • 浅阴影只给 selected 或浮层
文案
  • 使用“确认 / 修订 / 拒绝”
  • 禁用态必须说明原因
  • 不使用完成式过度声明
  • Chat 提案不等于执行

5. 下方工作台 Tab

下方区域不是杂物栏，它承载 Chat、提案、运行日志、证据、调试和系统建议。

运行日志查看事件流和失败session、turn、station、timestamp跳节点、复制 trace；无 handoff 禁用 retry

调试开发者/管理员诊断API、BFF route、DTO snapshot、denylist复制 debug bundle；写操作禁用
建议理解下一步安全动作阻断原因、建议修复、owner、priority创建提案；无目标时禁用

多技能审查结论

UI、UX、研究和无障碍角度一致认为：方向可进入 V12-0P，但实现前必须固化交互变体。
