允许与禁止

本阶段最重要的是防止 false green：好看的原型不能被当作真实实现证据。

本阶段允许
  • 组件拆分和 HTML/CSS wireframe 审查
  • 讨论工作台布局、信息架构、视觉层级
  • 记录逐组件 bounded acceptance 和 open questions
  • 进入 V12-0P 高保真原型规划
本阶段禁止
  • 直接进入真实浏览器实现
  • 把 wireframe 当作 BFF、runtime 或 Figma 证据
  • 声称已经完成 Xpert 级体验或完整 Studio
  • 跳过 V12-0P 和 readiness audit

用户态状态机

用户在工作台能看到的状态不应只是一句“API 健康”，而要覆盖环境、会话、画布、证据、运行和权限。

drawio evidence
状态域
  • API：healthy / degraded / offline / misconfigured
  • 会话：idle / thinking / proposal_ready / awaiting_confirmation / blocked
  • 画布：empty / read_only / node_hover / node_selected / inspecting
  • 证据：none / collecting / ready / redacted / failed
  • 运行：not_started / queued / running / completed / failed / cancelled
状态机文件
  • v12-user-state-machine.drawio
  • 用于 V12-0P 继续细化状态色、禁用原因、错误恢复和证据可见性。
  • 该 drawio 是设计说明，不是 runtime evidence。

用户交互路径

从“打开工作室”到“自然语言产生提案”，用户应该始终知道系统状态和安全下一步。

1进入工作室顶栏显示 workspace、项目、API、模型、证据状态。
2定位产品区左导航进入智能体、工作流、技能/MCP、证据、监控。
3查看画布只读画布展示节点、边、小地图和禁用原因。
4检查工位节点卡与 Inspector 解释角色、目标、工具、MCP、质量、证据。
5提出目标Chat Workbench 生成 proposal timeline，确认前不发布、不运行。

六个核心组件

每个组件都包含用户目标、状态、边界和 wireframe。请按组件逐个审查，而不是只看整页观感。

1. 全局顶栏

解决“我在哪、系统是否健康、下一步能不能安全执行”的问题。

bounded accepted
用户目标10 秒内识别当前上下文和安全下一步。
禁用态发布/运行必须解释需要 proposal 或确认。
状态枚举API、运行、证据、权限四类状态都在顶栏可见。

2. 左侧产品导航

把工作室入口从零散按钮整理为可理解的产品区域。

bounded accepted
用户目标快速定位智能体、工作流、技能/MCP、证据和监控。
状态选中、数量徽标、折叠、禁用态。
逻辑一级 rail 是产品域，二级导航是域内对象。

3. 画布工作台

建立接近 Xpert 工作台密度的只读画布基础，同时保留 V12 边界。

bounded accepted
用户目标理解当前工作流结构、曲线连接和节点选择。
边界V12 只读，新增/发布/运行禁用并说明原因。
交互hover/click 节点显示详情浮层和检查器入口。

4. Agent / Station 节点卡

让每个工位 Agent 的职责、模型、工具、技能、MCP 和证据一眼可审查。

bounded accepted
用户目标看懂 Agent 在这个工位负责什么。
IO展示多输入、多输出、端口和下拉详情。
边界仅展示 redacted refs，不声明 Agent executor ready。

5. 右侧检查器

选中节点后解释配置、策略、质量、证据和 blocked/disabled 原因。

bounded accepted
用户目标知道这个节点为什么能做或不能做某事。
边界Inspector 是 read-model，不构造 runtime truth。
下一轮细化tab、证据 drawer 和质量信息层级。

6. Chat Workbench + Proposal Timeline

让自然语言目标进入 proposal，而不是被误解为已经执行工作流。

bounded accepted
用户目标用独立 chatbox 管理目标、上下文和历史。
边界确认前不会 publish/run。
联动聊天历史、proposal timeline 和画布高亮需要 V12-0P 细化。
