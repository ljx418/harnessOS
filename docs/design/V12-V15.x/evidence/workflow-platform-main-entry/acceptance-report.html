目标架构与当前实现

目标架构要求工作流平台成为首入口，并将画布、WorkflowDiff、发布、运行、Human Gate、Evidence Review 与受治理 Agent/Tool/Skill/MCP 资源收敛到同一产品路径。当前实现复用 PV21 BFF DTO 作为工作流闭环，复用 PV20 BFF DTO 作为受治理执行证据，不新增绕过 BFF 的浏览器调用。

架构层目标当前实现
Browser entry默认进入 PV13 Light Studio 工作流平台。App.tsx 与 WorkflowStudioLayout.tsx 将根入口和 workflow-platform 映射到 V13EditableStudio。
Canvas / Workbench力感画布、节点、端口、连线、Inspector 和底部审查区可操作。V13EditableStudio.tsx 与 v13-editable-studio.css 提供 PV13 基线体验，并记录浏览器动作日志。
BFF / DTO浏览器只通过 BFF DTO route 与后端交互。/bff/v13/*、/bff/pv21/*、/bff/pv20/* 通过网络日志验证。
Runtime / Evidence运行、人工门禁、证据审查在同一工作台可理解。通过 PV21/PV20 compatibility DTO 生成 runtime inspect、evidence panel 和 executor integration 报告。
