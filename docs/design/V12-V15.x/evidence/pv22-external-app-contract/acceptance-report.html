目标架构与当前实现

目标架构：External UI -> App-owned BFF -> HarnessOS SDK -> Capability Token / Scope Binding -> Gateway RPC -> Evidence。

当前实现：reference app 前端只调用 /bff/*，BFF 通过 Python SDK 使用真实 /v1/rpc Gateway TestClient 路径完成 session、turn、artifact、events、approval smoke。

用户场景路径

  1. 外部应用选择 reference_app/demo/local scope。
  2. 服务端 BFF 使用 capability token 调用 SDK。
  3. SDK 创建 session、提交 turn、登记外部 artifact、订阅事件、处理人工审批。
  4. 浏览器只消费 BFF DTO 和 BFF event proxy，不直连 raw Gateway。
