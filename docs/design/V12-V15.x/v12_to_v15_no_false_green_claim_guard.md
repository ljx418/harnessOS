# V12-V15 No False Green Claim Guard

## Allowed Claims

- V12-V15 planning complete: Xpert-level frontend interaction and productization roadmap ready for review.
- V12 complete: product entity, browser workbench and canvas foundation ready for review.
- V13 complete: editable Workflow Studio pilot slice ready for review.
- V14 complete: governed extension ecosystem pilot ready for review.
- V15 complete: Xpert-level frontend interaction baseline ready for review.

## Forbidden Positive Claims

- Xpert parity complete
- production ready
- full production GA
- complete Workflow Studio ready
- Agent executor ready
- unrestricted plugin ecosystem ready
- production deployment ready
- observability production complete
- autonomous workflow editing ready
- Xpert clone complete
- Xpert frontend parity ready
- Xpert-level UX complete
- Xpert-quality canvas ready
- product-grade frontend complete
- production self-hosting ready
- 已完全追平 Xpert
- 已完整复刻 Xpert
- 生产可用
- 全面生产可用
- Xpert 级交互已完成
- 低代码画布已完成
- 产品级前端已完成
- 完整工作流工作台已完成
- Agent 执行器已完成
- 插件生态生产可用
- 生产级自托管已完成
- Xpert 前端体验已完全追平

## Safe Contexts

Forbidden phrases may appear only in this document, stop conditions, audit
questions, boundary explanations and drawio warning boxes. They must not appear
as release notes, dashboard status or completion claims.

## Stage Order Rule

V15 cannot claim baseline readiness unless V12, V13 and V14 evidence packages
exist and pass. Planning docs cannot satisfy implementation evidence.

## Reference Evidence Rule

Xpert reference screenshots, local deployment notes and source-code observations
may support planning and gap analysis. They cannot satisfy HarnessOS runtime,
Studio, plugin, observability or deployment acceptance.

## Canvas Claim Rule

V12 read-only canvas foundation must not be summarized as complete Workflow
Studio. V13 editable Workflow Studio pilot slice must not be summarized as
complete Workflow Studio ready or Xpert parity complete.
